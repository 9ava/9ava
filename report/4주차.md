## ✅ 데이터 교환 포맷 비교

| 항목             | JSON                  | XML                 | YAML                         |
| ---------------- | --------------------- | ------------------- | ---------------------------- |
| **가독성**       | 높음                  | 낮음                | 매우 높음                    |
| **형식**         | Key-Value 쌍          | 태그 기반           | 들여쓰기 기반                |
| **데이터 타입**  | 문자열, 숫자, 배열 등 | 문자열 위주         | 다양한 타입 지원             |
| **주석 지원**    | ❌ 없음               | ✅ 있음             | ✅ 있음                      |
| **무게/용량**    | 가벼움                | 무거움              | 매우 가벼움                  |
| **사용 예**      | 웹 API, JS 통신       | SOAP, 레거시 시스템 | 설정파일 (Docker, K8s 등)    |
| **시리얼라이징** | 내장 함수 지원        | 구현 복잡           | 라이브러리 필요 (ex. PyYAML) |

> **요약**
>
> - JSON은 웹 표준으로 가장 널리 사용됨
> - XML은 태그 기반으로 구조 표현은 좋지만 무겁고 복잡
> - YAML은 사람이 읽기 쉬워 설정파일에 적합함

## ✅ 시리얼라이징(Serialization) 개념

> **정의**:  
> 데이터를 파일, 네트워크 전송 등에 적합하도록 문자열이나 바이너리 형태로 변환하는 것

### ✔ 예시 (JavaScript)

```js
const user = { name: "Jack", age: 25 };
const jsonStr = JSON.stringify(user); // '{"name":"Jack","age":25}'
```

- 역직렬화: `JSON.parse(jsonStr)` → 객체 복원
- 다른 언어 예시:
  - Python: `pickle`, `json`
  - Java: `ObjectOutputStream`

## ✅ HTTPS와 SSL 인증서 기초

### 🔒 HTTP vs HTTPS 비교

| 항목         | HTTP      | HTTPS                    |
| ------------ | --------- | ------------------------ |
| **보안성**   | 없음      | SSL/TLS로 암호화         |
| **포트**     | 80        | 443                      |
| **URL 예시** | `http://` | `https://`               |
| **보안**     | 평문 전송 | 암호화 전송, 인증서 기반 |

---

### 🔐 SSL 인증서란?

- **SSL/TLS 인증서**는 웹사이트와 클라이언트 간의 통신을 **암호화**하고, **서버 신원**을 인증함
- 보안 웹사이트는 `https://`로 시작하고 자물쇠 표시 🔒가 나타남

#### 주요 구성 요소

- 발급자 (CA, 인증기관)
- 공개키/비밀키
- 인증된 도메인, 유효기간 등 정보 포함

#### 역할

- 데이터 암호화 (MITM 방지)
- 신원 인증
- 무결성 보장
